Title: BLE HID Keyboard lpm应用开发

1. 简介

本文介绍了如何使用键盘扫描驱动实现带低功耗功能的蓝牙遥控器开发。示例代码使用4x4矩阵键盘，基于蓝牙开发板，支持普通睡眠模式、深度睡眠模式。

2.原理图及接线

如上图所示为hid keyboard硬件参考设计，其中P14、P15、P18、P20为行扫描信号，P23、P25、P31、P32为列扫描信号

3. 应用开发

本章节介绍了HID Keyboard lpm示例程序的开发流程，键盘扫描驱动主要通过i/o扫描的方式获取按键值，这里主要介绍了驱动使用流程以及注意事项，ble相关内容可参考BLE HID Keyboard文档。

3.2 注意事项

键盘驱动对低功耗进行了支持，示例代码中展示了两种低功耗模式，第一种是普通睡眠模式，第二种是standby模式，有如下两个注意事项:

  • 为避免丢键，普通睡眠唤醒后进行进行键盘扫描

    xxxxxxxxxx
    /*睡眠唤醒恢复函数*/
    int pm_after_sleep_action()
    {
        csi_gpio_wakeup_sleep_action();
    ​
        csi_pinmux_wakeup_sleep_action();
    ​
        csi_usart_wakeup_sleep_action(0);
        /*若当前无按键按下，触发一次按键扫描，该行代码将触发app_main中的按键扫描部分*/
        if(!g_key.press_num) {
            /*释放扫描信号量*/
            aos_sem_signal(&g_wait_first_trigger);
        }
        return 0;
    }

    int app_main(int argc, char *argv[]){

  • 禁止/使能 standby

    xxxxxxxxxx
    static void conn_change(ble_event_en event, void *event_data)
    {
        evt_data_gap_conn_change_t *e = (evt_data_gap_conn_change_t *)event_data;
    ​
        if (e->connected == CONNECTED && e->err == 0) {
            g_conn_handle = e->conn_handle;
            LOGI(TAG, "Connected");
            led_set_status(BLINK_SLOW);
            ble_stack_security(g_conn_handle, 2);
            /*建立连接后，使能键盘standby,在没有按键操作一段时间后（20s）,设备将自动进入standby模式*/
            keyboard_enable_standby();
        } else {
            g_conn_handle = -1;
            LOGI(TAG, "Disconnected err %d", e->err);
            start_adv();
            /*断开连接后，保持设备广播，禁止设备进入standby*/
            keyboard_disable_standby();
            led_set_status(BLINK_FAST);
        }
    }

4. 例程运行

以蓝牙开发板为例，BLE HID Keyboard lpm示例程序的运行步骤为：

  • 连接串口调试工具, 配置为 波特率: 115200, 数据位: 8, 校验位: None, 停止位: 1, 流控: None
  • 按照原理图方法连接键盘和开发板
  • 开发板拨码开关拨至GND
  • 按RESET按键，复位开发板
  • 设备开始广播后，将在串口上输出‘adv start’信息
