蓝牙产测指令集说明

1. 简介

CH6121 BLE SDK提供了蓝牙产品（DUT）的基本测试和DTM测试功能，其中基本测试使用AT指令集，DTM测试使用符合DTM规范的命令集。本文将主要介绍产品测试指令以及如何启动DUT测试。

2. AT测试指令

2.1. 总览

2.2. AT+MAC

命令类型 命令格式 响应
测试命令 AT+MAC=? +MAC=xx:xx:xx:xx:xx:xx
查询命令 AT+MAC? MAC Address: 11:22:33:44:55:66
执行命令 AT+MAC=xx:xx:xx:xx:xx:xx OK
参数说明 xx:xx:xx:xx:xx:xx - MAC地址
示例 AT+MAC?
MAC Address: C0:B1:82:E3:3B:CC
OK
注意事项

2.3. AT+FREQOFF

命令类型 命令格式 响应
测试命令 AT+FREQOFF=? +FREQ_OFF='value'
查询命令 AT+FREQOFF? +FREQ_OFF=-200 Khz
执行命令 AT+FREQOFF=value OK
参数说明 value - 频偏值；范围: -200KHZ ~ 200KHZ,步长: 20KHZ
示例 AT+FREQOFF?
+FREQ_OFF=-200 Khz
注意事项

2.4. AT+SLEEP

命令类型 命令格式 响应
测试命令 AT+SLEEP=? +SLEEP='mode'
执行命令 AT+SLEEP=mode OK
参数说明 mode - 低功耗模式，0 - sleep模式；
示例 AT+SLEEP=0
OK
注意事项 1. 目前仅支持Sleep模式
2. 进入低功耗模式后，需要手动复位DUT，才能继续下一项测试

2.5. AT+RXMODE

命令类型 命令格式 响应
测试命令 AT+RXMODE=? +RXMODE='rx_time'
执行命令 AT+RXMODE=value START OK
OK
参数说明 value : 扫描时长，单位:毫秒
范围: 0 ~ 1800
注意事项 输出START OK时，表示开始SCAN；输出OK时，表示SCAN结束

2.6. AT+IREBOOT

命令类型 命令格式 响应
测试命令 AT+IREBOOT=? +IREBOOT='mode'
执行命令 AT+IREBOOT=mode OK
参数说明 mode - 重启模式， 0 - 正常模式
注意事项 目前仅支持正常模式

4. DUT测试API

CH6121 BLE SDK提供了DUT测试入口， 应用程序在板级初始化完成后，可直接调用该函数进入DUT测试。

​x
# 头文件包含
#include <ble_dut_test.h>
​
# API
void ble_dut_start(void);
​
void board_yoc_init(void)
{
    ...
    
    /* Disbale LPM if need enter DUT mode */
    disableSleepInPM(0xFF);
​
    /* Enter DUT mode */
    ble_dut_start();
}
​

5.注意事项

  • DUT相关函数不放RAM上会影响灵敏度测量结果。
