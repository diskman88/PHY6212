概述

串行接口 (Serial Interface) 是指数据一位一位地顺序传送，其特点是通信线路简单，只要一对传输线就可以实现双向通信（可以直接利用电话线作为传输线），从而大大降低了成本，特别适用于远距离通信，但传送速度较慢。一条信息的各位数据被逐位按顺序传送的通讯方式称为串行通讯。串行通讯的特点是：数据位的传送，按位顺序进行，最少只需一根传输线即可完成；成本低但传送速度慢。串行通讯的距离可以从几米到几千米；根据信息的传送方向，串行通讯可以进一步分为单工、半双工和全双工三种。

接口定义

打开串口

xxxxxxxxxx
dev_t *device_open(const char *name);

根据指定的串口设备名，找开串口。在 YoC 中串口的设备名的命令规范为 "uart<id>", id 为串号的序号，从 0 开始。例如 "uart0" 表示0号串口设备。串设备打开成功能，返回设备的指针，如果打开失败，则返回 NULL。注意：设备打开后，在不使用时需要调用 device_close 函数关闭已打开的设备，否则会造成内存泄漏，也会影响到设备的低功耗设计。 串口设备的操作过程中，不保证多任务的同步性，使用者在使用时，在自行确定多任务之间的相互冲突问题。

  • 参数:

    • name: 串口设备名

  • 返回值:

    • 打开成功返回设备指针，打开失败返回NULL

关闭串口

xxxxxxxxxx
int device_close(dev_t *dev);

关闭已打开的串口设备。设备关闭后，如果返回出错误后，不能重试，否则会造成不可预期的问题。

  • 参数:

    • dev: 串口设备指针

  • 返回值：

    • 0：关闭成功，小于0，表示关闭出错，返回错误号如下：

      • -EBADFD 错误的设备指针
      • -EIO IO 操作错误
      • -EBADF 错误的设备指针(dev)

    • 0: 配置成功， 小于0 配置失败，返回错误号如下：

      • -EBADFD 错误的设备指针
      • -EINVAL 无效的参数
      • -EIO 错误的硬件操作

缓冲区大小设置

xxxxxxxxxx
int uart_set_buffer_size(dev_t *dev, uint32_t size);

设备串口驱动内部缓冲区大小，在串口运行时，改变串口内部缓冲大小可能会造成内部缓冲区数据的丢失。

  • 参数:

    • dev: 串口设备指针
    • size: 指定内部缓冲区大小

  • 返回值:

    • 0: 设置成功，小于0则设置失败，返回错误号如下：

      • -EBADFD 错误的设备指针
      • -ENOMEM 内存不足

同步发送数据

xxxxxxxxxx
int uart_send(dev_t *dev, const void *data, uint32_t size);

向已打开的串口设备同步发送数据。直到数据全部发送完全后返回。

  • 参数：

    • dev: 串口设备指针
    • data: 待发送数据缓冲区指针
    • size：待发送数据长度

  • 返回值：

    • 0 表示发送成功，小于则发送失败，返回错误号如下：

      • -EBADFD 错误的设备指针
      • -EINVAL 无效的参数

接收数据

xxxxxxxxxx
int uart_recv(dev_t *dev, void *data, uint32_t size, unsigned int timeout_ms);

从打开的串口设备中接收数据，并将数据存在规定大小的缓冲区中，收满数据或者超过指定的时间，则函数返回。

  • 参数：

    • dev: 串口设备指针
    • data: 待接收的数据缓冲区指针
    • size：待接收数据的最大长度

  • 返回值：

    • 大于等于0，表示接到的实际数据长度，小于表示接收出错，返回错误号如下：

      • -EBADFD 错误的设备指针
      • -EINVAL 无效的参数

设备事件回调函数

xxxxxxxxxx
void uart_set_event(dev_t *dev, void (*event)(dev_t *dev, int event_id, void *priv), void *priv);

设置串口事件的处理函数。事件处理函数格式如下：

xxxxxxxxxx
void (*event)(dev_t *dev, int event_id, void *priv);

event_id 为事件号，如下：

  • USART_EVENT_SEND_COMPLETE: 发送数据完成
  • USART_EVENT_RECEIVE_COMPLETE: 串口收到远程发送过来数据后调用本事件

注意：事件回调函数在驱动的中断中执行，所以用户使用时要格外小心，要保证在事件回调函数中不能使用引起任务调度的操作，如malloc/free, aos_mutex_lock/aos_mutex_unlock, 等。

  • 参数：

    • dev: 串口设备指针
    • event: 事件的处理函数
    • priv：指定事件的处理函数参数中的priv值

  • 返回值：无

xxxxxxxxxx

生产串口配置的默认值，默认值如下：

static void uart_event_callback(dev_t* dev, int event_id, void *priv)
{
    uservice_t *srv = (uservice_t *)priv;

        uart_set_event(uart, uart_event_callback, srv);
