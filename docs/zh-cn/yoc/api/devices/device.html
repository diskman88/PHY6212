Title: 设备驱动API

概述

设备驱动API接口是用来统一管理设备驱动的核心接口，包括设备的注册，注销，操作等。

接口定义

注册设备驱动

xxxxxxxxxx
int driver_register(driver_t *drv, void *config, int idx);

注册设备驱动。

  • 设备驱动结构体
xxxxxxxxxx
typedef struct _driver_obj driver_t;
​
struct _driver_obj {
    slist_t node;
    char   *name;
    char   *type;
    int16_t ref;
    int16_t device_id;
​
    dev_t * (*init)(driver_t *drv, void *config, int id);
    int     (*uninit)(dev_t *dev);
    int     (*lpm)(dev_t *dev, int state);
    int     (*open)(dev_t *dev);
    int     (*close)(dev_t *dev);
};

  • 参数

    • drv:设备驱动结构体指针
    • config:设备配置结构体指针
    • idx:设备ID

  • 返回值

    • 等于0:注册成功
    • 非０：注册失败

注销设备驱动

xxxxxxxxxx
int driver_unregister(const char *name);

注销设备驱动。

  • 参数

    • name:设备名字

  • 返回值

    • 等于0：注销成功
    • 其他：注销失败

设备查找

xxxxxxxxxx
dev_t *device_find(const char *name, int id);

查找设备。

  • 参数

    • name:设备名字
    • id:设备ID

  • 返回值

    • NULL：查找失败
    • 非NULL：查找成功，返回设备对象指针

锁住设备

xxxxxxxxxx
int device_lock(dev_t *dev);

锁住设备防止其他对象对该设备访问。

  • 参数

    • dev:设备结构体指针

  • 返回值

    • 0：锁设备成功
    • 非0：锁设备失败

释放设备

xxxxxxxxxx
int device_unlock(dev_t *dev);

释放设备允许其他对象对该设备访问。

  • 参数

    • dev:设备结构体指针

  • 返回值

    • 0：锁设备成功
    • 非0：锁设备失败

判断设备名是否有效

xxxxxxxxxx
int device_valid(dev_t *dev, const char *name);

判断设备名是否有效。

  • 参数

    • dev:设备结构体指针
    • name:设备名字

  • 返回值

    • 0：设备名字有效
    • 非0：设备名字无效

获取设备ID

xxxxxxxxxx
int device_tags(dev_t *dev);

返回设备ID。

  • 参数

    • dev:设备结构体指针

  • 返回值

    • 大于等于0：设备ID
    • 其他值：获取失败

打开设备

xxxxxxxxxx
dev_t *device_open(const char *name);

返回设备驱动对象。

  • 参数

    • name:设备名字

  • 返回值

    • NULL：打开设备失败
    • 非NULL:打开设备成功，返回设备驱动对象指针

打开指定ID的设备

xxxxxxxxxx
dev_t *device_open_id(const char *name, int id)

返回设备驱动对象。

  • 参数

    • name:设备名字
    • id:设备ID

  • 返回值

    • NULL：打开设备失败
    • 非NULL:打开设备成功，返回设备驱动对象指针

关闭设备

xxxxxxxxxx
int *device_close(const char *name);

返回设备驱动对象。

  • 参数

    • name:设备名字

  • 返回值

    • 0：关闭设备成功
    • 非0：关闭设备失败
